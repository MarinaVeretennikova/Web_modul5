15. Зачем нужен Advanced Direct Remailer?
 


Дальше стоит читать только тем, кто хочет настроить SMTP сервер для отправки почты. Программа Advanced Direct Remailer (ADR) сочетает в себе многофункциональный почтовый ремайлер, позволяющий доставлять исходящую почту прямо на почтовый сервер получателя в обход сервера провайдера. Программа имеет встроенный почтовый сервер SMTP и работает под Windows 9* / Me, Windows NT 4, Windows 200* / XP; под управлением Windows NT / 200* / XP может работать в качестве сервиса.

В ADR есть свой внутренний почтовый сервер SMTP и любую программу можно научить отправлять письма через ADR. Достаточно изменить в настройках Вашей программы всего одну строчку! Подробное описание настройки почтовых программ приведено в справке программы.

 
16. Установка Advanced Direct Remailer
 


Скачать программу можно с официального сайта http://www.massmail.ru/. К сожалению ADR является условно бесплатной. Если Вы хотите пользоваться этой программой без ограничений, то Вам требуется зарегистрировать (купить) ее.

Приступим.

  1. Загрузите программу.

  2. Разархивируйте adrr.zip и запустите файл setup.exe

  3. ADR можно будет установить в любую папку, в нашем случае это - С:\Program Files\Advanced Direct Remailer.

  4. После инсталляции в трее появится иконка в виде синей стрелки

    Для того чтобы открыть окно программы достаточно кликнуть левой кнопкой мышки по этой иконке.

  5. Теперь нужно настроить программу, для этого нажмите на кнопку Отправка и в выпадающем меню выберите Настройка...

  6. В окне настройки выберите закладку "Задания" в которой нужно настроить вариант доставки почты:

    • Вручную через меню программы - Не использовать автоматический запуск, запускать отправку только вручную.
    • Когда установлено соединение удаленного доступа - Запускать отправку при установленном выбранном соединении. Если выбрано “Любое соединение” – то при любом установленном соединении удаленного доступа. Отправка прекращается, если соединение разрывается.
    • Когда почта есть в "Исходящих" - Автоматический запуск отправки почты, как только письма появляются в папке "Исходящие". Для тех, у кого постоянное подключение к Интернет.

  7. Теперь откройте вкладку "Доставка":

    Потоки отправки (0..399) - Число одновременно отправляющихся писем. Увеличивая число потоков, Вы увеличиваете сетевой трафик и нагрузку на машину. Рекомендуется 15..25 потоков для модема на 33.6k.

    Сервер - Сервер исходящей почты (SMTP) Вашего провайдера. Может быть задан доменом или адресом IP. Примеры: smtp.isp.com, 195.2.83.112. Особенность: если указано NULL вместо имени сервера (для программистов: в данном случае это строка, а не указатель ;-), тогда письмо, при попытке доставить его через сервер провайдера, сразу попадет в папку “Плохие”. Например, совместно с настройкой “Доставлять через провайдера после X попыток” Вы можете выбрасывать сообщения, которые не удалось доставить за X попыток; совместно со списком задержек Вы сможете заблокировать некоторые почтовые домены. Вторая особенность: Вы можете задать до пяти серверов провайдера через запятую (например, “smtp.isp.com, mx1.isp.com, mx2.isp.com»). Обратите внимание, что все сервера должны иметь одинаковый порт и одинаковые настройки аутентификации.

  8. Вкладка "Сервер":

    TCP порт сервера - В нашем случае это 25 порт, который использует SendMail.

  9. Теперь нужно проверить, как всё работает, для этого скачайте небольшой скрипт (1,69 Кбайт) и установите его в дирректорию D:\cgi-bin\mailto.cgi. Теперь отправьте письмо с помощью этого скрипта, для этого наберите в своём браузере ссылку http://127.0.0.1/cgi-bin/mailto.cgi.
